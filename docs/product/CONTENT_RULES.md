# Content Rules

## Purpose

These rules control how knowledge is treated inside the Fencepac Knowledge Platform MVP.

## Core principle

The platform can organise and display knowledge, but it does not replace engineering judgement, project-specific drawings, supplier documentation, supervisor direction, or formal company approval.

## Status meanings

### `needs_review`

Content has been seeded, drafted or imported but has not been reviewed.

### `reviewed`

Content has been checked for MVP purposes and can be treated as stronger internal knowledge, but it still does not override project-specific documents.

### `superseded`

Content has been replaced, outdated or should not be used as current guidance.

### `draft`

Content is incomplete or experimental.

## Source rules

- Every evidence note should link to a source record.
- Supplier brochures are external references unless company-reviewed.
- Public webpages are external references unless company-reviewed.
- Project drawings marked draft or not-for-construction must not be treated as construction authority.
- Real project photos should be approved before public deployment.
- Temporary third-party images are acceptable only for local MVP testing.

## Training rules

- Training content is for learning and onboarding support.
- Training modules do not replace formal competency assessment.
- Quiz and checklist progress in the MVP is local only and not a formal record.
- Future versions should save learner progress against authenticated users.

## Safety rules

Escalate when:

- Site conditions differ from drawings or source assumptions
- Drainage details are unclear
- Foundation or bearing conditions are uncertain
- Surcharge near the wall crest is unclear
- Geogrid or reinforcement details are unclear
- Drawings or documents conflict
- A note is marked draft, needs review or superseded
- A claim affects safety, design, installation sequence or QA acceptance

## Public deployment rule

Before public or coworker-wide deployment:

- Replace unlicensed images
- Confirm source ownership and permissions
- Add authentication and roles
- Add audit trail
- Add proper reviewer workflow
- Add company approval process
